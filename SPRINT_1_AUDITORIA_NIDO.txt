Estado general del sprint: técnicamente casi listo para cierre. Se aplicaron las correcciones prioritarias del Sprint 1 sobre persistencia, validación de precio, UX del error de foto, límite de tags y cobertura mínima de tests.

Problemas corregidos:
- Persistencia crítica del alta: el repositorio ya no dispara el guardado en background sin confirmación. La publicación se persiste antes de continuar el flujo exitoso.
- Validación de precio: ahora el formulario exige precio numérico y mayor a 0. Se bloquean vacío, texto, `0` y negativos.
- UX del error de foto: el mensaje de foto requerida ya no aparece desde el primer render; solo después de intentar publicar.
- Límite de tags: se agregó un máximo de 8 tags por publicación manteniendo deduplicación case-insensitive e ignorando vacíos.
- Estabilidad del formulario: la previsualización de imágenes del selector ahora tiene `errorBuilder` para no degradar la UI ante fallos de red.

Qué quedó resuelto:
- El flujo de publicación espera el guardado persistente antes de mostrar éxito.
- El formulario bloquea publicaciones inválidas por precio.
- La UX inicial del formulario deja de mostrar un error prematuro de imagen.
- El sistema de tags tiene una protección básica de MVP.
- El proyecto cuenta con tests mínimos orientados a persistencia, rehidratación, múltiples publicaciones, validación de precio, límite de tags, deduplicación e ignorado de vacíos.

- Por eso la única validación pendiente para declarar cierre pleno es correr la suite y una verificación manual rápida de reinicio real de app en un entorno con SDK Flutter instalado.
